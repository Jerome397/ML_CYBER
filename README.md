# Cybersecurity ML Streaming Demo

## Overview

This project builds a complete machine learning pipeline for cybersecurity traffic classification.

It includes:

* Training multiple models: Logistic Regression, Random Forest, and Neural Network
* Hyperparameter tuning
* Model selection and promotion
* Real-time manual-triggered prediction simulation
* Backend API using FastAPI
* Interactive visualization using Streamlit
* Docker support for easy execution and deployment

The system supports three datasets:

* Binary classification: Attack vs Benign
* 7-class classification
* 33-class classification

## Important Note About Artifacts

The `artifacts/` folder is excluded from GitHub because it contains large generated trained model files.

However, the public Docker image on Docker Hub contains the required working project files and model artifacts needed to run the demo.

If the project is cloned from GitHub without the `artifacts/` folder, the artifacts can be regenerated by:

## Training

```bash
py ml\train_logistic_regression_binary.py
py ml\train_random_forest_binary.py
```

### Multiclass Models

```bash
py ml\train_logistic_regression_multiclass.py
py ml\train_random_forest_multiclass.py
py ml\train_mlp_multiclass.py
```

### 33-Class Models

```bash
py ml\train_logistic_regression_33class.py
py ml\train_random_forest_33class.py
```

---

## Hyperparameter Tuning

Neural network tuning:

```bash
py ml\tune_mlp_multiclass.py
```

This creates multiple runs stored in:

```bash
artifacts/runs/mlp_tuned_...
```

Each run keeps:

* Parameters used
* Validation scores
* Test scores

---

## Model Promotion

After training, the best model for each dataset can be selected and promoted.

### Binary

```bash
py ml\promote_best_binary_model.py
```

### Multiclass

```bash
py ml\promote_best_multiclass_model.py
```

### 33-Class

```bash
py ml\promote_best_33class_model.py
```

These scripts:

* Compare models using macro F1-score
* Copy the best model into the correct `artifacts/final_*` folder
* Prepare the final models used by the demo application

## Backend

The backend is built using FastAPI.

### Start Backend Locally

```bash
py -m uvicorn demo_app.backend:app --reload
```

### Available Endpoints

* `/health` → check system status
* `/tick` → process one data row and predict
* `/state` → get prediction logs
* `/reset` → reset logs
* `/dashboard` → get model metrics for visualization

## Demo Functionality

### Page 1 — Model Comparison

This page shows metrics for each model, including:

* Accuracy
* Macro F1-score

It also explains why a specific model was selected.

### Page 2 — Manual Prediction Logs

The demo contains a button called:

```text
Process Next Line
```

Each click:

1. Takes one row from the dataset
2. Transforms it into features
3. Sends it to the model
4. Displays the prediction result

The result includes:

* Predicted label
* Actual label
* Correctness
* Confidence

Tables are available for:

* Binary model
* Multiclass best model
* Multiclass neural network
* 33-class model

---

## How Prediction Works

1. A row is read from the test dataset
2. Features are extracted using `feature_order.json`
3. The trained model predicts the label
4. The label is mapped using `label_mapping.json`
5. The result is stored and displayed in the Streamlit interface

## Running the Project with Docker

This project can be run using Docker.

Docker allows the whole project to run inside a container without manually installing all Python dependencies.

---

## Prerequisites

Before running the Docker version, make sure you have:

* Docker Desktop installed
* Docker Desktop running
* A web browser

To check that Docker is installed correctly, open PowerShell and run:

```bash
docker --version
```

You can also test Docker with:

```bash
docker run hello-world
```

---

## Installing Docker on Windows

1. Download Docker Desktop for Windows from the official Docker website.
2. Run the installer as Administrator.
3. During installation, keep the option **Use WSL 2 instead of Hyper-V** enabled.
4. Restart the computer after installation.
5. Open Docker Desktop and wait until it is fully running.

If Docker is not recognized in PowerShell, close PowerShell, reopen it, and try again:

```bash
docker --version
```

---

## Important Windows Note

If Docker Desktop installation fails because of a permission issue related to:

```bash
C:\ProgramData\DockerDesktop
```

run the Docker Desktop installer again as Administrator:

```text
Right click on Docker Desktop Installer.exe → Run as administrator
```

If needed, open PowerShell as Administrator and remove the broken Docker folder:

```powershell
Remove-Item "C:\ProgramData\DockerDesktop" -Recurse -Force
```

Then run the Docker Desktop installer again as Administrator.

## Run the Project from Docker Hub

After pulling the image, run the container:

```bash
docker run -p 8501:8501 -p 8000:8000 jerome123k/ml-cyber:latest
```

Then open the Streamlit frontend in your browser:

```bash
http://localhost:8501
```

The FastAPI backend will be available at:

```bash
http://localhost:8000
```

You can check the backend health endpoint using:

```bash
http://localhost:8000/health
```

---

## Build the Docker Image Locally

You can also build the Docker image from the GitHub project.

First, clone the repository:

```bash
git clone YOUR_GITHUB_REPOSITORY_LINK
cd Project-ML-cyber
```

If the `artifacts/` folder is missing because it is excluded from GitHub, regenerate it first by running the training and promotion scripts.

Then build the Docker image:

```bash
docker build -t ml-cyber .
```

Run the local Docker image:

```bash
docker run -p 8501:8501 -p 8000:8000 ml-cyber
```

Then open the Streamlit application:

```bash
http://localhost:8501
```

---

## Push the Docker Image to Docker Hub

After building the image locally, log in to Docker Hub:

```bash
docker login
```

Tag the image:

```bash
docker tag ml-cyber jerome123k/ml-cyber:latest
```

Push the image:

```bash
docker push jerome123k/ml-cyber:latest
```

After pushing, anyone can run the project using:

```bash
docker pull jerome123k/ml-cyber:latest
docker run -p 8501:8501 -p 8000:8000 jerome123k/ml-cyber:latest
```

---

## Docker Notes

The container exposes two ports:

* `8501` for the Streamlit frontend
* `8000` for the FastAPI backend

Make sure Docker Desktop is running before executing any Docker command.
